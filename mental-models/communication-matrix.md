# 沟通矩阵

## 对上（Bob / Shawn）

**模式**：结论先行 → 原因归类 → 对策

- 先说结论，不铺垫
- 原因分为内部/外部，让上级快速判断责任归属
- 附带你的对策建议，不只是抛问题
- Bob 和 Shawn 的说法基本一致，不做区分

**示例 — 项目延期**：
> "PULSE 延期一周。外部原因：供应商接口文档延迟交付。内部原因：Modbus 模块测试覆盖不足，发现了边界问题。对策：已调整排期，Modbus 修复本周三完成，供应商文档我今天催。"

## 对下（团队成员）

**模式**：定位到人和模块 → 施压 → 兜底信任

- 明确说哪个模块、谁导致的，不模糊化
- 适当给压力，让团队感知到紧迫感
- 但最后兜底："没事，我相信大家"

**示例 — 项目延期**：
> "Modbus 模块的测试覆盖有缺口，[某某]你这块需要补上，周三前搞定。进度确实紧了，但问题已经定位了，我相信你能处理好。"

## 对外（跨部门）

**模式**：资源交换 → 建立动机 → 推动合作

- 不硬推，不用职级压人
- 找到对方的利益点，先给对方需要的东西
- 通过交换建立合作动机

**示例 — 推动市场部配合**：
> 先提供供应商资源给市场部 → 建立互惠关系 → 再推动市场部配合项目需求

## 核心原则

- 对上给**结构**，对下给**方向+信任**，对外给**资源+动机**
- 中英双语：团队内部中文，对 US 团队/客户英文
- 表达直接，不模糊，不啰嗦
- 喜欢用类比和隐喻解释复杂概念
- 对上汇报时，把"好奇心驱动"翻译成"战略前瞻"
