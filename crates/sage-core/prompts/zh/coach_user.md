## 深度分级规则（严格执行）

为每条洞察分配深度等级，标准如下：

- **episodic（情景）**：必须包含具体日期、人物或具体事件。示例："2026-03-20 向 Bob 提交了 PULSE 提案"
- **semantic（语义）**：行为模式或特征，但不含因果推理。示例："倾向于先行动后反思"或"沟通风格简洁"
- **procedural（程序）**：必须同时具备三个部分——"当 [情境] → [行动] 因为 [原因]"。若缺少任一部分，降级为 **semantic**。示例："当复杂度上升时，优先砍范围而非增加抽象层 —— 因为出货优于完美"**这是最常被错误分配的等级。**
- **axiom（公理）**：此处不要分配 axiom —— 只有 Evolution 才能晋升为 axiom

**验证规则**：输出每条洞察前，先核查：
- 如果你标注为 procedural，它是否明确包含情境触发、行动和原因三个部分？若否，改标为 semantic。
- 如果你标注为 axiom，改为 procedural 或 semantic —— 此处永远不分配 axiom。

请输出你新发现的核心洞察（每条一行，简洁）。规则：
1. 只输出新发现或需要更新的认知，不要重复已有内容
2. 每条认知以「行为模式：」「决策倾向：」「沟通偏好：」等前缀开头
3. 每条简洁一行，不要写长段落
4. 只输出洞察内容，不要其他解释
